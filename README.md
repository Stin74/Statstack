# StatStack

Daily baseball stats puzzle. Pick 3 all-time career leaders per stat whose ranks sum as close as possible to today's target number.

---

## Quick start

### 1. Generate the data files

```bash
pip install pandas requests
python build_data.py
```

This pulls from the [Chadwick Bureau Lahman database](https://github.com/chadwickbureau/baseballdatabank) and writes five JSON files to `data/`:

| File | Stat | Source column |
|------|------|---------------|
| `data/hr.json` | Career Home Runs | `Batting.HR` |
| `data/rbi.json` | Career RBIs | `Batting.RBI` |
| `data/hits.json` | Career Hits | `Batting.H` |
| `data/pitcher_k.json` | Career Strikeouts | `Pitching.SO` |
| `data/pitcher_w.json` | Career Wins | `Pitching.W` |

Each file is a JSON array of 1,000 objects: `[{"rank":1,"name":"Barry Bonds","value":762}, ...]`

### 2. Deploy to Cloudflare Pages

1. Push this folder to a GitHub repo
2. Go to [Cloudflare Pages](https://pages.cloudflare.com/) → Create a project → Connect to GitHub
3. Select your repo — **no build command needed**, output directory is `/` (root)
4. Click Deploy

Cloudflare serves `index.html` and the `data/` files statically. The `_headers` file configures caching automatically.

## Keeping rankings current

The Lahman database updates periodically as new seasons conclude. To refresh rankings automatically:

### GitHub Actions (recommended)

The workflow at `.github/workflows/refresh-data.yml` runs `build_data.py` every morning at 7 AM UTC, commits any changes, and pushes — which triggers a new Cloudflare Pages deploy automatically.

You don't need to configure anything beyond pushing the workflow file to your repo.

### Manual refresh

```bash
python build_data.py
git add data/
git commit -m "Refresh rankings"
git push
```

## How the puzzle works

- Five MLB career stat categories are shown each day
- For each category, pick 3 players whose all-time career ranks **sum as close as possible** to the target number
- Example: if the target is 500 and you pick the #1, #200, and #299 all-time home run leaders, your sum is 500 — perfect
- Score per category = `|rank_sum − target|`
- Final score = sum of all 5 category penalties (lower is better, 0 is perfect)
- The target number is the same for all players on a given day

## Project structure

```
statstack/
├── index.html                       Game (single file, no build step)
├── build_data.py                    Data builder (run once, then nightly)
├── data/
│   ├── hr.json                      Generated by build_data.py
│   ├── rbi.json
│   ├── hits.json
│   ├── pitcher_k.json
│   └── pitcher_w.json
├── _headers                         Cloudflare cache rules
└── .github/workflows/
    └── refresh-data.yml             Nightly data refresh
```

## Extending to other sports

When ready to add NFL, NBA, or NHL:
1. Write a data builder for that sport (similar pattern to `build_data.py`)
2. Add a `data/<sport>/` folder with stat JSON files
3. Add a sport selector to `index.html` and update the `STATS` config and seed logic
